## 2. 핵심 메커니즘 (역기획)

### 2.1 코어 루프

```
[트리거]            [액션]              [보상]                  [투자]
매시 정각 푸시 알림 → 2~4초 클립 촬영   → 친구들의 '같은 시간'    → 그룹 유지, 아카이브 축적,
(놓치면 끝,           + 짧은 캡션         클립 확인 +              내일도 참여할 동기
 사후 촬영 불가)      (편집 없음)         하루 끝 자동 완성 브이로그
```

- **일일 사이클**: 새벽 4시 ~ (다음날) 새벽 4시 기준으로 하루가 리셋되고, 사이클 종료 시 클립이 시간순으로 자동 합성됨
- **클립 길이**: 2~4초 — "찍는 부담"을 의도적으로 0에 수렴시킴
- **그룹 크기**: 기본 4명, 요청 시 확장 (최대 12명까지 분할화면 합성)
- **솔로 모드**: 친구 초대 없이 개인 영상 일기로도 사용 가능

### 2.2 설계 원칙 3가지

1. **강제된 동시성 (Enforced Simultaneity)** — 모두가 같은 시각에 찍는다. "지금 친구는 뭘 하고 있을까"라는 호기심이 매시간 리텐션 훅이 됨.
2. **무편집·실시간 원칙 (No-edit, No-backdating)** — 사전 녹화 불가, 사후 보정 불가. 큐레이션 피로에 지친 Z세대에게 "It feels more real"이라는 가치를 제공.
3. **프라이빗 그룹 (No Audience)** — 공개 피드·팔로워·좋아요 수 없음. 보는 사람이 초대된 친구뿐이므로 "잘 보여야 한다"는 압박이 사라짐.

### 2.3 기능 인벤토리

| 기능 | 설명 |
|---|---|
| 정시 알림 촬영 | 매시간 푸시 → 카메라 즉시 실행 → 2초 촬영 + 캡션 |
| 자동 브이로그 합성 | 사이클 종료 시 시간순 자동 편집, 그룹이면 분할화면 |
| 그룹(로그) 생성/초대 | 연락처·아이디 기반 친구 초대, 솔로 로그 가능 |
| DM + 리액션 | 친구 로그에 짧은 반응/메시지 |
| 저장/내보내기 | 완성 브이로그 갤러리 저장 (저장 허용 설정으로 상대 저장 제어) |
| 색상/꾸미기 | 분할화면 내 개인 색상 선택 등 가벼운 개인화 |

### 2.4 온보딩 플로우

1. 이메일/전화/소셜 로그인 → 2. 프로필 설정 → 3. 카메라·마이크·사진·**알림** 권한 (알림이 코어라 필수) → 4. "친구와 함께 vs 혼자 기록" 선택 → 5. 로그 생성/초대

### 2.5 성장 엔진

- **내부 루프**: 그룹 앱이므로 1명이 쓰려면 친구 3명을 데려와야 함 → 초대가 곧 활성화 (K-factor 내장)
- **외부 루프**: 완성된 세로 분할화면 브이로그가 그 자체로 인스타 릴스/틱톡용 콘텐츠 → 워터마크 달고 외부 확산 → 신규 유입 ("인스타보다 재밌어" 류의 UGC 바이럴)

### 2.6 약점 (우리가 공략할 틈)

1. **알림 피로**: 매시간 푸시는 강력하지만 이탈 사유 1순위 (방해금지 모드와 충돌)
2. **리텐션 절벽 위험**: BeReal과 동일한 구조 — 신선함이 떨어지면 "오늘도 비슷한 일상"의 한계. 콘텐츠의 다양성이 사용자의 일상 다양성에 종속됨
3. **수익모델 부재**: 무료 + 인앱결제 없음
4. **안정성/완성도**: 평점 3.5, 저장 실패·알림 누락 등 품질 이슈 다수
5. **솔로 가치 약함**: 혼자 쓰면 그냥 영상 일기 — 그룹이 깨지면 앱도 버려짐
